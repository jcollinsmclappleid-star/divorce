uick-add assumption tiles across the wizard
Goal
Add tap-to-apply assumption tiles on key wizard steps so users can complete intake quickly with sensible UK starting figures, then refine individual fields below. This pattern is proven on our RedundancyCalculatorUK wizard — adapt it for divorce/separation modelling.

Success criteria: A user can reach a meaningful preview/report in under 5 minutes by tapping presets instead of typing every number from scratch.

UX pattern (must follow)
Callout panel at top of each quick-add section:

Title: e.g. “Quick-add typical UK costs” or “Common post-separation assumptions”
One-line helper: “Tap a tile to use a typical figure as a starting point — adjust any field below to match your situation.”
Left border accent (border-l-4 border-rose-300 for expenses, border-cyan-300 for income)
Tappable tile grid (grid-cols-2 sm:grid-cols-3 gap-2):

Icon (lucide) + short label
Inactive: ~£X/mo typical in muted text
Active: live value in bold tabular nums with category colour; tile gets border-gold/50 bg-card shadow-sm
Inactive hover: hover:border-primary/30 hover:bg-muted/60
Behaviour:

First tap → apply preset median/patch to form state
Second tap on active tile → open inline edit OR focus the matching field below (pick one; inline edit preferred for expenses)
Tiles never replace manual entry — full inputs/table remain below
Disclaimer copy on every assumption block: planning figures only, not advice, not CMS/legal/tax assessment.

Brand styling (DivorceCalculatorUK)
Navy text: #1a3357
Gold active state: border-gold/50, text-gold for “+ Add” hints
Muted panels: bg-muted/40
Match existing wizard components (wizard-shell, inline-confirm flash on save if available)
