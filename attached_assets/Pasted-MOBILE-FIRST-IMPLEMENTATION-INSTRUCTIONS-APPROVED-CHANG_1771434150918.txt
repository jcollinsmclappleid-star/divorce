MOBILE-FIRST IMPLEMENTATION INSTRUCTIONS (APPROVED CHANGES + SPECIFIC DEFAULTS)

Context:
Assume the majority of users will be on mobile (375px width). The current results and report UI requires excessive vertical scrolling, reducing readability and conversion.

Objective:
Reduce scroll fatigue by ~50–70% on mobile WITHOUT removing analytical content or undermining the professional “financial brief” tone. Implement changes in the main UI (results pages) and ensure the report view remains printable but also readable on mobile.

IMPORTANT:
Proceed to implement the items below (approved). Do NOT implement tabs or swipe gestures at this stage.

========================================================
A) INFORMATION ARCHITECTURE (MOBILE-FIRST GROUPING)
========================================================

Restructure each scenario detail view into 3 grouped blocks (this improves mental flow and reduces scroll):

1) FINANCIAL SUSTAINABILITY (keep highly visible)
   - Stability Score (headline + classification)
   - Capital Runway headline (Party A / Party B summary line)
   - Key risk flags (funding shortfall, affordability benchmark exceed, depletion year)
   - Liquidity ratio line (Liquid capital ÷ annual expenses)

2) CAPITAL POSITION (expandable detail)
   - Source of Funds (summary-first with expandable breakdown)
   - Net equity + selling costs clarity
   - Final allocated capital position (A/B)

3) HOUSING POSITION (expandable detail)
   - Affordability benchmark assessment (modelled)
   - Buyout requirement + funding gap (if applicable)
   - Housing transition note (if sell scenario)

After these blocks, keep additional analysis collapsible:
- Monthly Snapshot (expandable)
- Projection chart/table (expandable)
- Sensitivity snapshot (expandable)
- Negotiation levers (expandable)
- Scenario considerations (expandable)

========================================================
B) DEFAULT EXPANSION RULES (CRITICAL FOR MOBILE UX)
========================================================

In ScenarioDetailCard, default states on mobile:

EXPANDED by default:
- Scenario narrative summary (short + structured)
- Stability Score headline (and classification)
- Capital Runway summary line (Party A and Party B)
- Key flags summary (max 3 items)

COLLAPSED by default (with clear “Show details”):
- Source of Funds breakdown (show totals only)
- Monthly Snapshot (show surplus/deficit only)
- Housing Feasibility panel (show “Within/Exceeds benchmark” only)
- Projection chart/table
- Sensitivity snapshot
- Negotiation levers
- Scenario considerations
- Assumptions (UI)
- Glossary (UI)

Rationale:
Never hide the “conclusions”. Hide the “detail”.

========================================================
C) COLLAPSIBLE SUB-SECTIONS WITHIN SCENARIO DETAIL (APPROVED)
========================================================

Implement collapsible subsections inside ScenarioDetailCard using existing CollapsibleSection pattern.

Each collapsible section must:
- Have a clear title
- Show 1–2 summary lines when collapsed
- Use professional label language (no casual phrasing)
- Avoid heavy animations

Recommended section list + collapsed summaries:

1) Source of Funds (Capital Construction)
   Collapsed summary: “Party A liquid capital: £X | Party B liquid capital: £Y”
   Expand: full breakdown table

2) Monthly Sustainability Snapshot
   Collapsed summary: “Party A surplus/deficit: £X/mo | Party B: £Y/mo”
   Expand: income/expense/mortgage rows

3) Housing Feasibility (Modelled Benchmarks)
   Collapsed summary: “Benchmark: Within / Exceeds thresholds”
   Expand: ratio, income multiple, mortgage payment

4) Capital Projection (5-Year)
   Collapsed summary: “Lowest capital (A): £X | (B): £Y”
   Expand: table + chart (if available)

5) Sensitivity Snapshot
   Collapsed summary: “+1% rate: £X/mo | +10% costs: -£Y/yr”
   Expand: full sensitivity detail

6) Negotiation Levers
   Collapsed summary: “3 suggestions available”
   Expand: list of lever suggestions

7) Scenario Considerations
   Collapsed summary: “Key considerations available”
   Expand: 3–5 structured prompts
