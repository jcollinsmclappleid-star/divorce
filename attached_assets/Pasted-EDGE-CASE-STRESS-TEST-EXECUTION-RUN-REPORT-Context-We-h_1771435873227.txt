EDGE-CASE STRESS TEST EXECUTION (RUN + REPORT)

Context:
We have a UK divorce financial modelling app with:
- Scenario modelling (Sell / A Keeps / B Keeps)
- Split sliders (assets + pensions)
- Stability score + classification
- Capital runway projection (5 years)
- Affordability benchmark assessment
- PDF report generation

Task:
Run the edge-case stress tests below through the app/model and report results back to me in a structured table + findings summary.

IMPORTANT RULES:
1) Do NOT refactor or “improve” the model during testing.
2) If a case fails, record it, explain why, propose the minimal fix, and stop there (no silent changes).
3) Ensure UI outputs and report outputs reconcile (where applicable).
4) For each case, test Sell scenario + A Keeps + B Keeps (if applicable). If a scenario cannot apply (e.g., no mortgage), report “N/A” with explanation.
5) Verify invariants and flag any breach.

========================================================
A) REQUIRED INVARIANTS (CHECK FOR EVERY CASE)
========================================================
For each scenario and party, verify:
- No NaN / Infinity / undefined values
- No divide-by-zero errors (especially ratios)
- Currency formatting consistent (GBP)
- Totals reconcile:
  PartyA liquid + PartyB liquid + fees/known deductions == total distributable liquid pool (within rounding tolerance)
- Runway logic:
  If capital hits <= 0 in year N, runway badge must show Year N (per party)
- Stability logic:
  Stability deductions must match trigger conditions
- Affordability logic:
  Mortgage-to-net-income ratio is computed on same time basis (monthly/monthly or annual/annual) and stays plausible (no 3000% anomalies unless income extremely small and time basis correct)
- Preview vs paid outputs match exactly
- UI vs PDF values match exactly

========================================================
B) OUTPUTS TO CAPTURE (FOR EACH CASE)
========================================================
For each scenario (Sell, A Keeps, B Keeps), capture:
- Net equity
- Combined net asset pool
- Party A starting liquid capital
- Party B starting liquid capital
- Funding shortfall (if any)
- Mortgage payment (if any)
- Mortgage-to-net-income ratio (if any)
- Monthly surplus/deficit (A, B)
- Runway outcome (A, B): Sustained 5y or Yr N depletion
- Stability score (A, B)
- Stability classification (A, B)
- Any warnings/flags displayed
