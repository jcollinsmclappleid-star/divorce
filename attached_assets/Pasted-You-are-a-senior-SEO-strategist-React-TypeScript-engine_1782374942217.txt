You are a senior SEO strategist, React/TypeScript engineer, UK divorce-finance UX copywriter, and conversion-rate optimisation specialist.

I am building **DivorceCalculatorUK.co.uk**.

The site already has a broad sitemap with many guide pages, including divorce financial settlement, pensions, property, mediation, maintenance, financial disclosure, clean break orders, consent orders, and related divorce finance topics.

The issue:
The site is still underweight on **exact calculator-intent SEO pages**. It has many guide-style pages, but users searching for calculators are closer to converting because they want a tool, not just information.

Your job:
Assess the current routes/pages/sitemap first, then build a stronger **calculator-intent SEO cluster** that feeds users into the existing divorce calculator and paid full report flow.

Do not blindly create duplicate pages.
Do not overwrite strong existing pages.
Do not create thin doorway pages.

The new pages must be distinct, useful and search-intent specific.

Core positioning:
DivorceCalculatorUK is a private, assumption-based divorce financial modelling tool for England and Wales. It helps users model settlement scenarios involving property, pensions, savings, investments, debts, income, housing and cashflow.

It is not:

* legal advice
* financial advice
* tax advice
* pension advice
* mortgage advice
* investment advice
* a prediction of court outcomes
* a replacement for a solicitor, mediator, financial adviser, pension specialist or mortgage adviser

Use careful wording:

* “model”
* “scenario”
* “under these assumptions”
* “may”
* “could”
* “planning model”
* “private report”
* “does not predict court outcomes”

Avoid:

* “you are entitled to”
* “the court will award”
* “you should accept”
* “best settlement”
* “guaranteed”
* “legal advice”
* “financial advice”
* “pension advice”
* “tax advice”
* “mortgage advice”

PHASE 3 — Page structure requirements

Each new page must include:

1. SEO hero

* exact keyword in H1
* concise explanation
* CTA above fold

2. Calculator/tool CTA
   Use the existing calculator route/report flow.
   Do not create a separate broken calculator if the current engine already exists.

3. “What this can model”
   Use bullets specific to the page.

4. “What this cannot decide”
   Include non-advisory boundaries specific to the page.

5. Example scenario
   Create one realistic example using illustrative numbers.
   Make clear it is an example only.

6. Internal links
   Add related calculator links and guide links.

7. FAQ section
   Minimum 4 FAQs per page.
   Use FAQPage schema.

8. Breadcrumbs
   Add BreadcrumbList schema.

9. Disclaimer
   Use this or similar:

“DivorceCalculatorUK is an assumption-based financial modelling tool. It does not provide legal, financial, tax, pension, mortgage or investment advice and does not predict court outcomes.”

10. CTA to paid report
    After useful free content, include:

“Unlock the full private settlement report”

Do not put the paid CTA before useful content.

PHASE 5 — Canonical strategy

Do not canonical all calculator pages to one page unless they are near duplicates.

Preferred approach:
Make each new page unique enough to index.

Use:

* `/free-divorce-calculator-uk` for free-entry search intent.
* `/divorce-settlement-calculator-uk` for main commercial calculator search.
* `/divorce-calculator-uk` for generic exact-match calculator search.

PHASE 7 — Conversion requirements

Each calculator page should route to the monetisation flow.

Conversion flow:

1. User lands on calculator-intent SEO page.
2. User gets useful explanatory content.
3. User clicks into calculator or starts modelling.
4. User sees free/basic output.
5. User is offered paid full private report.
6. If relevant, user is offered paid written clarity review.

Do not make the page feel like a hard-sell landing page.
The paid report CTA should feel like the natural next step.

Suggested CTA copy:

* “Start free calculator”
* “Model my settlement”
* “Compare scenarios”
* “Unlock the full private report”
* “Get a written clarity review”
